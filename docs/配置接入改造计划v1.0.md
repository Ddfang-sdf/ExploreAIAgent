## 0. 核心约束

**严禁改动任何已有测试用例代码。** 所有现有构造函数的签名和行为必须保持不变。

改造策略：每个模块在保留原有构造函数的基础上，新增一个 `from_config` 构造函数或配置注入方法。

## 1. ApiAdapter

### 当前状态

```rust
pub fn new(api_mode: ApiMode) -> Self        // 测试使用
pub(crate) async fn invoke_llm(...) -> ...    // stub 返回 Err
const MAX_RETRIES: usize = 3;                  // 硬编码
// 无 api_key / base_url / model
```

### 改造内容

| 改造项 | 改造方式 |
|:---|:---|
| `MAX_RETRIES` 常量 | 新增 `max_retries` 构造参数，保留原常量作为默认值 |
| `api_key` / `base_url` / `model` | 新增字段，新增 `from_config(&LlmConfig)` 构造函数 |
| `invoke_llm` | 用 `reqwest` 实现真实 HTTP 调用，读取 `self.base_url/api_key/model` |

### 改造后的构造函数

```rust
// 原有签名，不变 —— 测试代码零改动
pub fn new(api_mode: ApiMode) -> Self {
    ApiAdapter {
        api_mode,
        max_retries: MAX_RETRIES,
        base_url: String::new(),
        api_key: String::new(),
        model: String::new(),
    }
}

// 新增 —— 生产环境使用
pub fn from_config(config: &LlmConfig) -> Self {
    let api_mode = match config.api_mode.as_str() {
        "responses" => ApiMode::Responses,
        _ => ApiMode::Chat,
    };
    ApiAdapter {
        api_mode,
        max_retries: config.max_retries,
        base_url: config.base_url.clone(),
        api_key: config.api_key.clone(),
        model: config.model.clone(),
    }
}
```

### 测试影响

**零改动。** `ApiAdapter::new(ApiMode::Chat)` 在所有测试中保持兼容。`new()` 构造的实例不含 api_key，`invoke_llm` 行为保持不变（无 api_key 时返回 Err）。

## 4. ExplorationContextTool

### 当前状态

```rust
pub const EXPLORATION_TOKEN_THRESHOLD: usize = 5500;
pub const EXPLORATION_TOKEN_TARGET_RATIO: f64 = 0.70;
pub const MIN_REMAINING_RECORDS: usize = 5;
pub const RECORD_MAX_CHARS: usize = 8000;

pub fn new(session_id: String) -> Self
```

### 改造内容

| 改造项 | 改造方式 |
|:---|:---|
| 四个常量全部 | 新增字段 + `set_thresholds()` 方法或 `from_config()` |

### 改造后的结构体和方法

```rust
pub struct ExplorationContextTool {
    context: Mutex<ExplorationContext>,
    backup_summary: Mutex<Option<ExplorationSummary>>,
    // 新增配置字段
    token_threshold: usize,
    token_target_ratio: f64,
    min_remaining_records: usize,
    record_max_chars: usize,
}

// 原有签名，不变 —— 测试零改动
pub fn new(session_id: String) -> Self {
    ExplorationContextTool {
        context: Mutex::new(...),
        backup_summary: Mutex::new(None),
        token_threshold: EXPLORATION_TOKEN_THRESHOLD,      // 保持不变
        token_target_ratio: EXPLORATION_TOKEN_TARGET_RATIO,  // 保持不变
        min_remaining_records: MIN_REMAINING_RECORDS,        // 保持不变
        record_max_chars: RECORD_MAX_CHARS,                  // 保持不变
    }
}

// 新增 —— 生产环境：Orchestrator 构造 ECT 后调用
pub fn configure(
    &mut self,
    exploration: &ExplorationConfig,
    context: &ContextConfig,
) {
    self.token_threshold = exploration.token_threshold;
    self.token_target_ratio = exploration.token_target_ratio;
    self.min_remaining_records = context.min_remaining_records;
    self.record_max_chars = context.record_max_chars;
}
```

内部逻辑中 `EXPLORATION_TOKEN_THRESHOLD` 常量引用替换为 `self.token_threshold` 字段，其余同理。

### 测试影响

**零改动。** 测试使用 `ExplorationContextTool::new("session-id")`，构造函数内部仍使用原有常量值。

## 9. 改造汇总

| 模块 | 改造方式 | 新增方法 | 测试改动 |
|:---|:---|:---|:---|
| ApiAdapter | `from_config(&LlmConfig)` + 实现 `invoke_llm` | 1 个构造函数 + 1 个方法 | **零** |
| Orchestrator | `from_config(...)` 多 1 个参数 | 1 个构造函数 | **零** |
| DeepExplorer | `from_config(&DeepExplorerConfig)` | 1 个构造函数 | **零** |
| ExplorationContextTool | `configure()` 设置阈值 | 1 个方法 | **零** |
| ExecuteShellTool | `from_config(PathBuf, &ToolsConfig)` | 1 个构造函数 | **零** |
| SearchStrategyAgent | `from_config(...)` 多 1 个参数 | 1 个构造函数 | **零** |
| ConversationManager | 无需改动 | 0 | **零** |
| ConversationContextTool | `configure(&ConversationConfig)` | 1 个方法 | **零** |

**所有 8 个模块，测试代码零改动。** 总新增接口 7 个 `from_config`/`configure` 方法 + ApiAdapter 的 `invoke_llm` 实现。

## 10. 生产启动入口

```rust
// main.rs 或 lib.rs 的启动入口
let config = AppConfig::load(None)?;

let llm_adapter = Arc::new(ApiAdapter::from_config(&config.llm));
let tool_registry = Arc::new(ToolRegistry::new(PathBuf::from(&config.workspace.path)));
let cm = ConversationManager::new(ApiAdapter::new(ApiMode::Chat)); // CM 的 adapter 后续也可接 config

let mut ect = ExplorationContextTool::new(session_id);
ect.configure(&config.exploration, &config.context);

let mut cct = ConversationContextTool::new(session_id);
cct.configure(&config.conversation);

let orchestrator = Orchestrator::from_config(
    llm_adapter.clone(),
    tool_registry.clone(),
    cm,
    ect,
    &config.exploration,
);

orchestrator.run(question, session_id).await
```
